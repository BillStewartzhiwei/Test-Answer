# 答测小程序功能文档

## 1. 产品定位

答测是一款轻量级答题小程序，面向「创建者出题、参与者答题」的场景。创建者可以创建答题空间、发布题目套、生成邀请码；参与者通过邀请码加入空间后完成答题，并查看成绩、答题详情和排行榜。

当前版本支持无后端本地运行，数据保存在微信开发者工具本地缓存中，便于快速演示完整业务闭环。

## 2. 角色说明

### 创建者

创建者用于组织答题活动，主要能力包括：

- 注册时选择创建者身份。
- 创建答题空间。
- 管理空间内的题目套。
- 创建、编辑、发布、关闭题目套。
- 生成邀请码邀请参与者加入空间。
- 手动失效或删除邀请码。
- 查看单套题排行榜。
- 查看空间累计排行榜。

### 参与者

参与者用于加入空间并完成答题，主要能力包括：

- 注册时选择参与者身份。
- 输入邀请码加入指定空间。
- 查看已加入空间列表。
- 查看空间内已发布题目套。
- 进入答题页完成作答。
- 提交后查看得分、正确率、用时和当前排名。
- 查看每道题的对错详情。
- 查看单套题排行和空间总排行。

角色在注册后不可更改。

## 3. 当前运行模式

### 本地无后端模式

当前小程序默认使用本地 mock API，不需要启动后端服务。

本地数据存储在微信缓存中，主要缓存键包括：

- `mockDb`：本地业务数据。
- `token`：模拟登录 token。
- `user`：当前登录用户。
- `lastResult`：最近一次答题结果。

适用场景：

- 微信开发者工具快速演示。
- 无 Java、Maven、MySQL 环境时体验流程。
- 产品原型验证。

### 后端预留模式

项目中已包含 Spring Boot 后端工程，后续可切换为真实服务端接口。

后端技术栈：

- Spring Boot 2.7
- MyBatis-Plus
- JWT
- MySQL

数据库表：

- `users`
- `spaces`
- `invite_codes`
- `space_members`
- `quizzes`
- `questions`
- `options`
- `quiz_attempts`
- `answers`

## 4. 功能模块

### 4.1 登录与注册

当前版本使用测试 `openId` 进行本地登录。

流程：

1. 用户进入登录页。
2. 输入测试 openId，例如 `creator001` 或 `user001`。
3. 如果是新用户，进入注册页。
4. 用户选择角色：创建者或参与者。
5. 输入昵称后完成注册。
6. 注册后进入首页。

说明：

- 本地模式不会调用 `wx.login`。
- 角色注册后不可修改。
- 不同 openId 可模拟不同用户。

### 4.2 首页

首页根据当前用户角色展示不同入口。

创建者首页：

- 展示「我创建的空间」列表。
- 支持进入空间详情。
- 支持创建新空间。

参与者首页：

- 展示「我加入的空间」列表。
- 支持进入空间详情。
- 支持通过邀请码加入空间。

### 4.3 空间管理

创建者可以创建答题空间。

空间信息包括：

- 空间名称。
- 空间描述。
- 空间状态。

当前小程序页面支持：

- 创建空间。
- 查看空间下的题目套。
- 进入邀请码管理。
- 创建题目套。
- 查看空间总排行。

后端工程中已预留：

- 编辑空间。
- 归档空间。

### 4.4 邀请码管理

创建者可以在空间内管理邀请码。

邀请码规则：

- 6 位字母数字码。
- 默认有效期 7 天。
- 默认最大使用次数 50 次。
- 一个空间可生成多个邀请码。

当前支持操作：

- 生成邀请码。
- 查看邀请码列表。
- 查看使用次数。
- 查看过期时间。
- 手动失效邀请码。
- 删除邀请码。

失效与删除区别：

- 失效：邀请码记录仍保留，但不能继续使用。
- 删除：邀请码从管理列表中移除。

### 4.5 加入空间

参与者通过邀请码加入空间。

流程：

1. 参与者进入「加入空间」页面。
2. 输入创建者提供的 6 位邀请码。
3. 系统校验邀请码是否有效。
4. 校验通过后加入对应空间。

限制：

- 只有参与者可以加入空间。
- 重复加入同一空间会提示错误。
- 已失效的邀请码不能使用。
- 使用次数达到上限的邀请码不能使用。

### 4.6 题目套管理

创建者可以在空间内创建题目套。

题目套信息包括：

- 标题。
- 答题时限。
- 总分。
- 状态。

题目套状态：

- 草稿：可编辑题目。
- 已发布：参与者可作答。
- 已关闭：停止作答。

当前支持：

- 创建题目套。
- 编辑草稿题目套。
- 发布题目套。
- 关闭题目套。
- 查看题目套排行。

### 4.7 题目编辑

当前支持三种题型：

- 单选题。
- 多选题。
- 判断题。

每道题支持：

- 设置题干。
- 设置分值。
- 添加选项。
- 标记正确答案。
- 删除题目。

题型规则：

- 单选题只能有一个正确答案。
- 多选题可以有多个正确答案。
- 判断题默认两个选项：正确、错误。

题目套总分会根据每道题分值自动累计。

### 4.8 答题

参与者只能作答已发布题目套。

答题页支持：

- 展示题目标题。
- 展示总分。
- 展示倒计时。
- 按题选择答案。
- 提交作答。

限制：

- 每套题默认只能作答一次。
- 如果设置了答题时限，页面会显示剩余时间。
- 超时后会自动提交。

### 4.9 成绩与答题结果

提交后立即展示答题结果。

结果信息包括：

- 本次得分。
- 题目总分。
- 正确率。
- 答题用时。
- 当前排名。
- 每道题对错情况。
- 用户选择的答案。
- 正确答案。

判分规则：

- 用户选择答案与正确答案完全一致时，该题得满分。
- 单选、判断、多选均按整题判分。
- 多选题少选、多选、错选均不得分。

### 4.10 排行榜

当前支持两类排行榜。

#### 单套题排行榜

展示某一题目套下所有参与者的排名。

排名字段：

- 名次。
- 昵称。
- 得分。
- 总分。
- 用时。

#### 空间总排行榜

展示某一空间内所有题目套的累计排名。

统计方式：

- 累计用户在该空间所有题目套中的得分。
- 累计总分。
- 累计答题用时。

排名规则：

1. 得分高者优先。
2. 同分时用时短者优先。

## 6. 演示流程

### 创建者流程

1. 登录页输入 `creator001`。
2. 注册为创建者。
3. 创建空间。
4. 进入空间详情。
5. 创建题目套。
6. 编辑题目并保存。
7. 发布题目套。
8. 进入邀请码管理。
9. 生成邀请码并分享给参与者。
10. 查看排行榜。

### 参与者流程

1. 登录页输入 `user001`。
2. 注册为参与者。
3. 点击加入空间。
4. 输入创建者的邀请码。
5. 进入空间详情。
6. 选择已发布题目套。
7. 完成答题并提交。
8. 查看得分、正确率、用时和排名。

## 7. 当前限制

当前版本为本地演示 MVP，存在以下限制：

- 数据只保存在本地缓存中，清缓存后会丢失。
- 不同设备之间数据不互通。
- 当前登录为测试 openId，不是真实微信授权登录。
- 本地版没有真实后端权限校验。
- 编辑空间、归档空间等能力已在后端预留，但小程序端暂未完整开放入口。

## 8. 后续可扩展方向

可继续扩展的功能包括：

- 接入真实微信登录。
- 切换到 Spring Boot 后端和 MySQL 数据库。

- 增加答题记录列表。
